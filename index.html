Imitation Learning for Xpilot

Training robust agents from multiple specialists using structured-state demonstrations - COM 407

This project focuses on imitation learning in the Xpilot environment. The primary goal is to record demonstrations from several specialist agents, train behavior cloning models, and evaluate whether a combined policy can outperform each specialist.

View Project Details View Proposal (PDF)

Overview

Motivation: Reinforcement learning in Xpilot is valuable but data intensive. A good imitation system for Xpilot is not yet established. The project studies whether combining demonstrations from agents with different specialties yields a better overall agent.

Objectives

  • Record demonstration data from multiple specialist agents (e.g. dodger, shooter etc).
  • Train behavior cloning models and a combined model using structured-state inputs.
  • Evaluate and compare performance of specialist agents and the combined agent.
Idea flow: Environment -> Structured State -> Demonstrations -> Behavior Cloning -> Evaluation

Methodology

Environment design

Xpilot is a 2D arena with agents, projectiles, powerups, and walls. Episodes terminate on agent death or a time limit. An environment wrapper will expose a normalized structured state vector and accept discrete actions.

Structured-state representation

Observations are fixed-length numeric vectors: agent state x, y, v, heading; enemies with relative distance, heading etc; bullets with dist, vel, heading; and wall distances.

Data collection and experts

I will create or use specialist agents to generate demonstrations: e.g. a dodger focused on survival, a shooter focused on accuracy. Each agent will run multiple episodes and log state-action pairs for training.
